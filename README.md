API gateway

Описание
API gateway - это единая точка входа для всех клиентских запросов. Вместо обращений к каждому микросервису клиент отправляет запрос только в gateway. Сервер определяет, какому сервису предназначен запрос, выполняет общие проверки и перенаправляет запрос в соответствующий сервис.

Такой подход скрывает внутреннюю архитектуру системы от клиента, упрощает взаимодействие с микросервисами и позволяет централизованно реализовать общую функциональность.

Основные функции:
1. Маршрутизация запросов к нужным микросервисам
2. Кэширование
3. Структурированное логгирование
4. Rate limiting для ограничения количества запросов 
5. Добавление CORS заголовков
6. Аутентификация и авторизация пользователей

Архитектура:
Клиент взаимодействует только с API Gateway. Gateway принимает запрос, выполняет общую обработку (логирование, проверку авторизации, rate limiting и другие сквозные функции), после чего перенаправляет запрос в соответствующий микросервис.

Без API Gateway:
Frontend ──┬── auth-service:8081/api/v1/auth/login
           ├── auth-service:8081/api/v1/users/me
           ├── books-service:8082/api/v1/books
           └── books-service:8082/api/v1/books/123/reviews

С API Gateway:
Frontend ──── gateway:8000/api/v1/* ──┬── auth-service:8081
                                      ├── books-service:8082
                                      └── worker-service
