# API Gateway

## Описание

API Gateway - это единая точка входа для всех клиентских запросов. Вместо обращения напрямую к каждому микросервису клиент отправляет все запросы только в gateway. Gateway определяет, какому сервису предназначен запрос, выполняет общую обработку и перенаправляет запрос в соответствующий сервис.

Такой подход скрывает внутреннюю архитектуру системы от клиента, упрощает взаимодействие с микросервисами и позволяет централизованно реализовать общую функциональность.

## Основные функции

- Маршрутизация запросов к нужным микросервисам
- Кэширование
- Структурированное логгирование
- Rate limiting для ограничения количества запросов
- Добавление CORS-заголовков
- Аутентификация и авторизация пользователей

## Архитектура

Клиент взаимодействует только с API Gateway. Gateway принимает запрос, выполняет общую обработку (логирование, проверку авторизации, rate limiting, кэширование и другие сквозные функции), после чего перенаправляет запрос в соответствующий микросервис.

### Без API Gateway

```text
Frontend ──┬── auth-service:8081/api/v1/auth/login
           ├── auth-service:8081/api/v1/users/me
           ├── books-service:8082/api/v1/books
           └── books-service:8082/api/v1/books/123/reviews
```

### С API Gateway

```text
Frontend ──── gateway:8000/api/v1/* ──┬── auth-service:8081
                                      ├── books-service:8082
                                      └── worker-service
```
